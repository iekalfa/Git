# Επίπεδο 2: Βασικές Λειτουργίες

## Στόχοι Μάθησης
- Προσθήκη αρχείων στο staging area
- Δημιουργία commits
- Παρακολούθηση της κατάστασης και του ιστορικού

## 📓 Σύνοψη
Εδώ μαθαίνουμε τις καθημερινές εντολές: πώς να προσθέτουμε αρχεία (add), να δημιουργούμε snapshots (commit), να βλέπουμε την κατάσταση (status), το ιστορικό (log) και τις διαφορές (diff). Αυτές οι εντολές είναι το 80% της δουλειάς σας με το Git.

## 🔑 Βασικές Έννοιες
- **Staging Area (Index)**: Προσωρινή περιοχή όπου "σκηνοθετείτε" τις αλλαγές πριν το commit.
- **Untracked**: Αρχεία που το Git δεν παρακολουθεί ακόμα.
- **Modified**: Αρχεία που άλλαξαν αλλά δεν είναι staged.
- **Staged**: Αρχεία έτοιμα για commit.
- **Committed**: Αρχεία αποθηκευμένα στο ιστορικό.
- **.gitignore**: Αρχείο που ορίζει ποια αρχεία να αγνοηθούν.

> Ροή: Working Directory → (`git add`) → Staging → (`git commit`) → History

## Θεωρία

### Ο Κύκλος Ζωής των Αρχείων
Τα αρχεία στο Git μπορούν να βρίσκονται σε διάφορες καταστάσεις:
- **Untracked**: Νέα αρχεία που δεν παρακολουθούνται ακόμα
- **Modified**: Αρχεία που έχουν αλλάξει αλλά δεν έχουν προστεθεί στο staging
- **Staged**: Αρχεία που είναι έτοιμα για commit
- **Committed**: Αρχεία που έχουν αποθηκευτεί στο ιστορικό

### Τι είναι το Staging Area;
Το staging area είναι μια ενδιάμεση περιοχή όπου προετοιμάζετε τις αλλαγές πριν τις κάνετε commit.

## Πρακτική Εφαρμογή

### Προσθήκη Αρχείων (git add)
Πριν καταγράψετε μια αλλαγή, πρέπει να επιλέξετε ποια αρχεία θέλετε να συμπεριλάβετε. Η εντολή `git add` μεταφέρει αλλαγές από το working directory στο staging area.

```bash
# Προσθήκη συγκεκριμένου αρχείου
git add README.md

# Προσθήκη όλων των αρχείων
git add .

# Προσθήκη όλων των αρχείων με συγκεκριμένη επέκταση
git add *.txt
```

### Αγνόηση Αρχείων (.gitignore)
Συχνά υπάρχουν αρχεία που δεν θέλουμε να παρακολουθεί το Git (π.χ. προσωρινά αρχεία, logs, compiled code). Αυτά τα ορίζουμε σε ένα ειδικό αρχείο με όνομα `.gitignore`.

```bash
# Δημιουργία αρχείου .gitignore
touch .gitignore
```

Παράδειγμα περιεχομένου `.gitignore`:
```text
# Αγνόηση όλων των .log αρχείων
*.log

# Αγνόηση φακέλου node_modules
node_modules/

# Αγνόηση αρχείων συστήματος
.DS_Store
Thumbs.db
```

### Δημιουργία Commit (git commit)
Το commit είναι σαν να τραβάτε μια φωτογραφία (snapshot) του project σας εκείνη τη στιγμή. Αποθηκεύει τις αλλαγές που βρίσκονται στο staging area μόνιμα στο ιστορικό του Git.

```bash
# Commit με μήνυμα
git commit -m "Προσθήκη αρχείου README"

# Commit όλων των τροποποιημένων αρχείων
git commit -am "Ενημέρωση αρχείων"

# Commit με αναλυτικό μήνυμα (ανοίγει editor)
git commit
```

### Έλεγχος Κατάστασης (git status)
Η εντολή `git status` είναι η πιο χρήσιμη εντολή για να καταλάβετε σε τι κατάσταση βρίσκεται το project σας. Σας δείχνει ποια αρχεία έχουν αλλάξει, ποια είναι staged και ποια είναι untracked.

```bash
# Δείτε την τρέχουσα κατάσταση
git status

# Συνοπτική προβολή
git status -s
```

### Προβολή Ιστορικού (git log)
Για να δείτε τι έχει συμβεί στο παρελθόν, χρησιμοποιούμε το `git log`. Εμφανίζει τη λίστα με τα commits, ποιος τα έκανε και πότε.

```bash
# Προβολή όλων των commits
git log

# Συνοπτική προβολή
git log --oneline

# Προβολή με γραφικά
git log --graph --oneline --all

# Προβολή των τελευταίων N commits
git log -n 5
```

### Προβολή Διαφορών (git diff)
Ενώ το `git status` σας λέει *ποια* αρχεία άλλαξαν, το `git diff` σας δείχνει *ακριβώς τι* άλλαξε μέσα σε αυτά (γραμμή προς γραμμή).

```bash
# Διαφορές μεταξύ working directory και staging
git diff

# Διαφορές μεταξύ staging και τελευταίου commit
git diff --staged

# Διαφορές συγκεκριμένου αρχείου
git diff README.md
```

## Ασκήσεις

1. Δημιουργήστε ένα νέο αρχείο `hello.txt` με το περιεχόμενο "Hello Git"
2. Δημιουργήστε ένα αρχείο `secret.log` και ένα αρχείο `.gitignore`
3. Ρυθμίστε το `.gitignore` ώστε να αγνοεί όλα τα `*.log` αρχεία
4. Ελέγξτε με `git status` ότι το `secret.log` αγνοείται
5. Προσθέστε το `hello.txt` και το `.gitignore` στο staging area
6. Κάντε commit με το μήνυμα "Προσθήκη hello.txt και gitignore"
7. Τροποποιήστε το αρχείο και δείτε τις διαφορές με `git diff`
8. Προσθέστε και κάντε commit τις αλλαγές
9. Δείτε το ιστορικό με `git log --oneline`

## Καλές Πρακτικές

- Γράφετε σαφή και περιγραφικά μηνύματα commit
- Κάντε commits συχνά αλλά με λογική
- Κάθε commit πρέπει να αντιπροσωπεύει μια ολοκληρωμένη αλλαγή
- Χρησιμοποιήστε το παρόν χρόνο στα μηνύματα (π.χ. "Προσθήκη" αντί για "Προστέθηκε")

## 🧪 Mini Quiz
1. Τι κάνει το `git add .`; (α) Commit όλα (β) Στέλνει όλα στο staging ✓
2. Πώς βλέπω τι άλλαξε σε ένα αρχείο; (α) `git diff` ✓ (β) `git status`
3. Το `.gitignore` αγνοεί αρχεία που; (α) Είναι ήδη committed (β) Δεν έχουν γίνει ποτέ add ✓

## ⚠️ Συνηθισμένα Λάθη
- `git add .` προσθέτει και αρχεία που δεν θέλετε (λύση: καλό `.gitignore`).
- Μηνύματα commit τύπου "fix" ή "update" χωρίς λεπτομέρειες.
- Ξεχνάτε να κάνετε commit μετά το staging (τα αρχεία μένουν staged).

## 💡 Συμβουλή
Χρησιμοποιήστε `git status` συχνά – είναι ο καλύτερος φίλος σας.

## 🔁 Ανακεφαλαίωση
`git status` → `git add` → `git status` → `git commit -m` → `git log`
